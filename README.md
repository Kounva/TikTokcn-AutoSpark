# 抖音火花助手

## 功能特性

### 账户管理
- 扫码登录（抖音 App 扫码授权）
- 手机号登录
- 手动登录（Base64 Cookie 方式）
- Cookie 一键导出
- 登录状态实时检测
- 上次登录 IP 记录
- 管理员密码修改

### 好友管理
- 好友列表展示（头像、火花天数）
- 好友搜索过滤
- 实时刷新好友数据
- 一键发送消息

### 聊天对话
- 类抖音原生聊天界面（气泡式消息展示）
- 查看好友聊天记录
- 直接发送文字消息
- 支持发送抖音内置表情包（通过 Selenium 自动化操作表情面板）

### 定时任务
- 为好友创建每日定时发送任务
- 支持自定义消息内容（留空使用每日名言）
- 修改已有任务执行时间
- 删除定时任务
- 最近任务快捷入口

### 首页看板
- 浏览器 / 登录状态监测
- 好友数量 / 定时任务数量统计
- 快速操作入口
- 系统运行信息（版本、在线时长、后端地址）

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | Vue 3 (Composition API，全局构建，无构建步骤) |
| UI 组件库 | Element Plus + Element Plus Icons |
| 路由 | Vue Router 4 |
| HTTP 客户端 | Axios |
| 后端框架 | FastAPI + Uvicorn |
| 浏览器自动化 | Selenium（支持 Edge / Chrome / Chromium / Brave） |
| 定时调度 | schedule |

**特点**：前端所有依赖（Vue / Element Plus / Axios）已本地化到 `dist/lib/`，无需 npm install，无需构建步骤，启动后端即可使用。

## 快速开始

### 环境要求

- **Python 3.8+**
- **任一受支持浏览器**：Microsoft Edge / Google Chrome / Chromium / Brave
- Windows / Linux

### 安装依赖

```bash
pip install -r requirements.txt
```

依赖清单：
- fastapi
- uvicorn
- selenium >= 4.10（内置 Selenium Manager，自动下载驱动）
- schedule
- requests

### 启动服务

```bash
python 抖音自动续火花-后端.py
```

启动后直接输出访问地址，无需任何交互：
- 默认显示浏览器窗口（无头模式易崩溃）
- 端口从 `config.json` 读取（默认 8080），被占用时自动切换到可用端口

启动后访问控制台输出的地址即可使用。

### Linux 运行说明

- 推荐浏览器：`Google Chrome` 或 `Chromium`
- 若浏览器已加入系统 PATH，可直接在「设置」页选择浏览器类型后保存
- 若浏览器不在默认路径，可在「设置」页填写自定义路径，例如：
  - `/usr/bin/google-chrome`
  - `/usr/bin/chromium`
  - `/snap/bin/chromium`
- Linux 服务器若无桌面环境，建议先开启图形环境或使用虚拟显示；无头模式可用，但稳定性通常不如有头模式

### 使用流程

1. 浏览器打开管理后台 → 用默认账号 `admin / 123456` 登录
2. 进入「设置」页确认浏览器类型、浏览器路径和显示模式
3. 首页点击「初始化浏览器」启动自动化环境
4. 扫码或手机号登录抖音
5. 登录成功后回到首页 → 刷新好友列表
6. 在「聊天」页选择好友 → 查看聊天记录、发送消息或表情包
7. 在「定时任务」页为好友添加每日续火花任务

## 端口配置

端口信息保存在 `config.json`：

```json
{
  "port": 8080,
  "show_browser": true,
  "browser_name": "chrome",
  "browser_path": "",
  "task_jitter_minutes": 8,
  "max_consecutive_failures": 3,
  "dedupe_window_days": 7,
  "message_templates": [
    "今天也要开心呀",
    "来给你续一下小火花"
  ]
}
```

- 启动时自动读取，被占用时自动切换到可用端口
- 在「设置」页可修改端口和浏览器显示模式，**重启后端后生效**
- `browser_name` 支持 `edge` / `chrome` / `chromium` / `brave`
- `browser_path` 留空时自动查找系统浏览器，填写后优先使用自定义路径
- `task_jitter_minutes` 为定时任务触发后的随机延迟分钟数
- `max_consecutive_failures` 为自动熔断阈值，连续失败达到该值后会暂停所有定时任务
- `dedupe_window_days` 为消息去重时间窗，尽量避免在该时间内给同一好友重复发送相同内容
- `message_templates` 为全局模板池；留空消息时会从模板池和每日名言中优先选择未重复内容
- 各浏览器会使用独立的 `*_user_data/` 目录保存用户数据，保持浏览器指纹一致，避免每次启动被视为新设备

## 防封号措施

为降低封号概率，本工具已内置以下优化：

1. **随机操作间隔**：发送消息、打开聊天、获取表情等操作之间使用随机延迟（1~2.5 秒），模拟真人操作节奏
2. **固定浏览器指纹**：使用本地浏览器用户数据目录持久化会话，避免每次启动都被识别为新设备
3. **降低登录检测频率**：前端 `GetLogin` 状态检查有 30 秒冷却时间，避免频繁请求触发风控
4. **随机延迟执行**：定时任务支持 `0 ~ N` 分钟随机延迟，避免每天固定整点触发
5. **模板池去重**：留空消息时优先从模板池中挑选近期未发过的内容，减少重复文案
6. **自动熔断暂停**：连续失败达到阈值后自动暂停全部任务，避免异常状态下持续触发风控

### 用户操作建议

- 定时任务建议设置在正常作息时段（如 20:00-23:00），避免凌晨自动发送
- 如果需要自定义消息，建议配置 3 条以上模板并用换行或 `||` 分隔，而不是长期固定一句话
- 不要同时登录多个设备操作同一账号
- 保持网络环境稳定，不要频繁切换 IP
- 仅用于续火花，不要频繁手动发送消息

## 常见问题

### 浏览器崩溃 / 会话断开
- 优先使用**有头模式**（设置页「浏览器显示」开关，默认开启）
- Linux 下优先选择 `Chrome` 或 `Chromium`，必要时在设置页填写浏览器可执行文件路径
- 崩溃后前端会提示「浏览器会话已断开」，点击「初始化浏览器」重启即可

### 登录状态不正确
- 点「设置」页的「刷新状态」按钮，会实际检查抖音页面登录状态
- 如需强制设置登录状态，可使用「强制登录」按钮（需输入密码确认，仅调试用）

### 端口被占用
- 启动时自动检测，若端口被占用会自动切换到下一个可用端口
- 控制台会打印实际使用的端口

### 定时任务丢失
- 定时任务会自动持久化到 `tasks.json`，重启后端并初始化浏览器后会自动恢复

### 自动任务被暂停
- 当连续发送失败达到阈值时，系统会自动暂停所有定时任务
- 可在「设置」页查看失败原因、今日发送次数和最近风险状态
- 处理完异常后，点击「恢复自动任务」即可继续运行
